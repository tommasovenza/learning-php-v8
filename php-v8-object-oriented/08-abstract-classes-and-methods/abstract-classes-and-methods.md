# Classi astratte e metodi astratti

L’astrazione è tipicamente implementata definendo una classe astratta oppure un’interfaccia che fornisce un insieme di metodi e attributi senza specificarne l’implementazione.
Definendo una classe astratta andremo a definire una sorta di template che verrà poi usato nelle classi derivate, in modo da creare una struttura omogenea e riciclare tutta una serie di funzionalità.

A differenza delle altre classi, quelle astratte non possono essere istanziate. Ciò significa che poter creare un oggetto di classe astratta dovremo avere una classe standard che estende la classe astratta.
Una classe astratta è usata per definire il cosa e non il come
